# 项目生产安全配置基线

## 密钥与配置

- 真实 API Key、JWT Secret、数据库密码、第三方 token 不允许提交到 GitHub。
- 本地密钥只能放在 `.env/`、系统环境变量或部署平台密钥管理中。
- `.env.example` 只能放占位符。
- 生产环境必须配置 `APP_API_KEY` 或更强的 JWT/网关鉴权。
- 生产环境必须显式配置 `CORS_ALLOW_ORIGINS`，不允许使用 `*` 搭配凭据。

## 认证与权限

- 后台接口默认需要登录或服务调用密钥。
- 管理能力需要校验角色或能力点。
- 用户只能访问自己有权限的资源。
- 登录失败、token 过期、权限不足应返回明确状态码，不泄漏内部实现。
- 退出登录或刷新 token 后，旧 token 的行为必须明确。

## 数据与隐私

- 响应中不得返回密码、密钥、完整 token、身份证、手机号等敏感信息。
- 日志中不得记录 Authorization、Cookie、API Key、明文密码或完整个人隐私数据。
- 测试数据应使用脱敏样例。
- 导出文件应限制字段范围，避免导出不必要的敏感数据。

## 文件与导出

- 文件名必须过滤 CR/LF、路径分隔符和特殊控制字符。
- 导出接口应限制格式、字段和数据量。
- 上传能力上线前必须限制文件类型、大小和解析方式。
- 不信任用户上传内容，不直接执行上传文件。

## AI 与 RAG

- RAG 知识库不应导入真实密钥、隐私样本或商业敏感原文，除非服务仅在可信内网运行。
- AI 输出必须经过结构校验，不直接信任模型返回。
- 生成失败应有明确错误，不暴露第三方 API Key 或完整上游响应。
- 日志可以记录模型名、耗时、检索条数，但不记录敏感 prompt 明文。

## 部署与运行

- 生产服务必须有健康检查。
- 生产日志应保留错误堆栈，但隐藏密钥和敏感请求头。
- 数据库、向量库、模型缓存、日志和 `.venv` 不进入版本库。
- 依赖升级后必须运行自动化测试。
- 公开部署前应验证鉴权、CORS、错误处理和导出安全。

## 最低上线检查

- `/health` 可访问。
- 业务接口无鉴权时被拒绝。
- CORS 只允许可信来源。
- `.env/`、`.model_cache/`、`data/chroma/`、`logs/` 未提交。
- 测试通过。
- 真实业务知识库已脱敏导入。
