# 个人项目验收标准模板

## 使用方式

当项目没有正式 PRD 或验收标准时，按模块补充本模板。每个模块至少保留“目标、范围、角色、核心规则、验收标准、非目标”六部分。

## 模板

### 模块名

填写模块名称，例如：认证与权限、内置问卷预测、自定义问卷、模型中心、分析预警、AI 测试用例生成。

### 目标

说明这个模块解决什么问题，以及用户完成后能得到什么结果。

### 范围

包含：

- 支持哪些页面、接口或任务。
- 支持哪些角色。
- 支持哪些输入、输出和状态。

不包含：

- 当前版本明确不做的功能。
- 需要后续迭代处理的能力。

### 角色与权限

列出可访问该模块的角色、能力点和限制：

- 匿名用户。
- 普通登录用户。
- 管理员。
- 仅后台或系统任务可访问的能力。

### 核心业务规则

列出必须遵守的规则：

- 必填字段和字段格式。
- 分值、阈值、状态流转。
- 数据归属和权限限制。
- 训练、发布、回滚、导出等关键约束。

### 验收标准

推荐使用 Given/When/Then：

```text
Given 前置条件
When 用户或系统执行某个动作
Then 系统应产生明确、可验证的结果
```

示例：

```text
Given 用户未登录
When 访问需要登录的后台接口
Then 系统应返回 401，且响应中不包含敏感数据
```

```text
Given 管理员已发布自定义问卷
When 匿名用户提交有效答案
Then 系统应保存响应记录，并能在后台响应列表中查询到该记录
```

```text
Given 训练任务处于运行中
When 用户请求任务详情
Then 系统应返回当前状态、进度、创建时间和可执行操作
```

### 异常验收

每个模块至少补充：

- 缺少必填参数。
- 参数格式错误。
- 资源不存在。
- 状态不允许。
- 权限不足。
- 重复提交。
- 外部依赖失败。

### 测试完成定义

一个模块达到可验收状态，需要满足：

- 核心成功路径有测试用例。
- 主要异常路径有测试用例。
- 权限边界有测试用例。
- 关键数据字段和状态流转有测试用例。
- 已知风险被记录到 `risks` 或缺陷文档中。
